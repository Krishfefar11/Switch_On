1. Requirement Validation & Gap Analysis [CONTEXT]
1.1 Missing Requirements Identified
No token refresh / expiry strategy — spec says JWT but never defines exp TTL or refresh flow. Decision: access token TTL = 15 min, refresh token TTL = 7 days stored in httpOnly cookie.
No rate-limiting — public /evaluate endpoint is unbounded. Decision: express-rate-limit 100 req/min per IP on evaluation routes.
No pagination on audit logs — unbounded query will OOM at scale. Decision: cursor-based pagination, default page size 50.
No soft-delete for flags — hard deletes destroy audit trail. Decision: add deletedAt field; DELETE route sets deletedAt, filters exclude soft-deleted records.
No environment support — single flag set means staging bleeds into prod. Decision: add environment field (development | staging | production) to FeatureFlag schema.
No flag versioning — spec mentions payload diff in audit logs but no version counter. Decision: add version integer that increments on every update.
Consumer demo app userId — spec says 'pass unique userId' but never explains origin. Decision: generate UUID on first visit, store in localStorage.
No CORS policy defined — cross-origin from demo app will be blocked. Decision: explicit CORS whitelist via env var ALLOWED_ORIGINS.
No health-check endpoint — required for container orchestration. Decision: GET /health returns { status: ok, uptime, dbStatus }.
Polling vs real-time — spec uses 2-second polling. This is acceptable for demo but wastes connections. Decision: upgrade consumer to Server-Sent Events (SSE); keep polling as fallback.
1.2 Security Risks in Original Spec
JWT secret hardcoded in examples — must come from env var JWT_SECRET (min 256-bit entropy).
bcrypt rounds not specified — use 12 rounds (balance of security vs latency at scale).
First-user-is-admin logic is a race condition under concurrent registrations — use MongoDB findOneAndUpdate with upsert to atomically count users.
Audit logs must be immutable — disable update/delete routes on AuditLog collection; use MongoDB capped collection or append-only index.
Evaluate endpoint exposes flag names publicly — add optional API key header X-Consumer-Key for consumer plane authentication.
2. Architecture Decisions [CONTEXT]
2.1 Monolith vs Microservices
Decision: Modular Monolith. Microservices add operational overhead (service mesh, distributed tracing, inter-service auth) that is unjustified for a team-sized feature flag tool. The codebase is structured in clearly separated layers (routes / controllers / services / models) so it can be split later with minimal refactoring. The evaluation engine is isolated as a pure function module so it can be extracted to a Lambda/Edge function when throughput demands it.
2.2 REST vs Event-Driven
Decision: REST for control plane CRUD + SSE for real-time push. Full event-driven (Kafka/RabbitMQ) is overkill for this scale. SSE gives real-time flag delivery to the consumer plane with a single persistent HTTP connection per client, no WebSocket upgrade handshake, and works through HTTP/2 proxies. The control plane uses REST because flag mutations are low-frequency, transactional operations that need strong consistency guarantees, not eventual consistency.

6. Feature Evaluation Engine [GENERATE]
File: src/services/evaluationEngine.js — Pure function, no side effects, no DB calls. Must be importable independently for unit testing.
// evaluationEngine.js
const crypto = require('crypto');
 
/**
 * Deterministically evaluates whether a user is in the rollout bucket.
 *
 * Algorithm:
 *   1. Concatenate featureName + ':' + userId
 *   2. Compute MD5 hash (fast, uniform distribution, not security-critical here)
 *   3. Take first 8 hex chars → parse as uint32
 *   4. bucket = (hashInt / MAX_UINT32) * 100  → float 0–100
 *   5. User is IN if bucket < rolloutPercentage AND flag.enabled === true
 *
 * Properties:
 *   - Same user always gets same bucket for a given flag name
 *   - Uniform distribution: ~1% of users per percentage point
 *   - Changing flag name = different distribution (intentional)
 *
 * @param {Object} flag - FeatureFlag document
 * @param {string} userId - Any stable string identifier
 * @returns {{ enabled: boolean, bucket: number, reason: string }}
 */
function evaluate(flag, userId) {
  if (!flag.enabled) {
    return { enabled: false, bucket: null, reason: 'FLAG_DISABLED' };
  }
 
  if (flag.rolloutPercentage === 100) {
    return { enabled: true, bucket: 100, reason: 'FULL_ROLLOUT' };
  }
 
  if (flag.rolloutPercentage === 0) {
    return { enabled: false, bucket: 0, reason: 'ZERO_ROLLOUT' };
  }
 
  const input  = `${flag.name}:${userId}`;
  const hash   = crypto.createHash('md5').update(input).digest('hex');
  const hashInt = parseInt(hash.substring(0, 8), 16);            // 0 – 4294967295
  const bucket  = (hashInt / 0xFFFFFFFF) * 100;                  // 0.0 – 100.0
 
  return {
    enabled: bucket < flag.rolloutPercentage,
    bucket:  parseFloat(bucket.toFixed(4)),
    reason:  bucket < flag.rolloutPercentage ? 'IN_ROLLOUT' : 'OUT_OF_ROLLOUT',
  };
}
 
module.exports = { evaluate };

7.8 SSE Service (src/services/sseService.js)
// In-memory client registry (use Redis pub/sub for multi-instance)
const clients = new Map();   // clientId → response object
 
function addClient(id, res) { clients.set(id, res); }
function removeClient(id)   { clients.delete(id); }
 
function broadcast(event, data) {
  const payload = `event: ${event}\ndata: ${JSON.stringify(data)}\n\n`;
  clients.forEach(res => {
    try { res.write(payload); }
    catch (e) { /* client disconnected, will be cleaned by 'close' event */ }
  });
}
 
module.exports = { addClient, removeClient, broadcast };
7.9 RBAC Middleware
// middlewares/rbac.js
const rbac = (allowedRoles) => (req, res, next) => {
  if (!req.user) return res.status(401).json({ error: 'Unauthorized' });
  if (!allowedRoles.includes(req.user.role)) {
    return res.status(403).json({ error: 'Forbidden: insufficient role' });
  }
  next();
};
module.exports = rbac;

  es.addEventListener(evt, e => {
    const { flag } = JSON.parse(e.data);
    if (evt === 'FLAG_DELETED') { delete flags[flag.name]; }
    else { flags[flag.name] = flag; }
    renderAll();
  });
});
 
es.onerror = () => console.warn('SSE disconnected, EventSource auto-reconnects');
 
// 3. Render — call /api/evaluate/:name?userId=<id> for each flag
async function evaluate(flagName) {
  const r = await fetch(`/api/evaluate/${flagName}?userId=${userId}&environment=development`);
  return r.json();
}
 
async function renderAll() {
  const container = document.getElementById('flags-container');
  container.innerHTML = '';
  for (const [name, flag] of Object.entries(flags)) {
    const result = await evaluate(name);
    const el = document.createElement('div');
    el.className = `flag-card ${result.enabled ? 'active' : 'inactive'}`;
    el.innerHTML = `
      <h3>${name}</h3>
      <p>${flag.description || ''}</p>
      <span class="badge">${result.enabled ? 'ENABLED FOR YOU' : 'DISABLED'}</span>
      <small>Rollout: ${flag.rolloutPercentage}% | Bucket: ${result.bucket ?? 'N/A'}</small>
    `;
    container.appendChild(el);
  }
}

10.2 Security Checklist
helmet() — sets X-Content-Type, X-Frame-Options, HSTS, CSP
express-mongo-sanitize — prevents NoSQL injection on all body/query params
express-rate-limit — 100/min on /evaluate, 10/min on /auth/login
bcrypt rounds=12 — never store plain passwords, never log passwords
JWT secret minimum 256 bits (32 random bytes, base64-encoded)
httpOnly + sameSite=strict on refresh token cookie
Audit log writes must never be skipped — wrap in try/catch that logs to stderr but never blocks the main response
Soft-delete flags: never expose deletedAt flags via public evaluate endpoint
Validate flag name as /^[a-z0-9_-]+$/ — prevents path traversal in evaluate URL
CORS: never use wildcard '*' — always use explicit ALLOWED_ORIGINS env var
10.3 Scalability Notes
Evaluation engine is stateless — safe to run on Lambda/Edge (extract to separate function later)
SSE sseService.clients is in-process memory — for multi-instance deploy, replace with Redis pub/sub (ioredis)
MongoDB: add replica set for HA; use read preference 'secondaryPreferred' for audit log reads
Flag caching: add Redis layer (TTL=5s) in flagService.getByName() to reduce DB reads on hot evaluate path
Rate limit store: use Redis adapter for express-rate-limit in multi-instance deploy
